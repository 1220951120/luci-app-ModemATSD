# luci-app-ModemATSD

面向 AK68 / 鲲鹏 C8 系列设备的 OpenWrt/ImmortalWrt 蜂窝模组管理插件，提供模组状态、网络与频段设置、短信、流量统计、调试、备份恢复和自动任务等 LuCI 功能。

> [!IMPORTANT]
> 本仓库是基于原作者项目进行维护和优化的衍生版本，不是原作者的官方发布仓库。

## 维护方向

当前分支主要围绕以下方向优化：

- **稳定性**：减少配置操作对模组和网络连接的不必要重启，完善错误处理与运行日志。
- **数据正确性**：使用 64 位安全的流量统计，修复累计流量超过 4 GiB 后溢出、实时速率异常等问题。
- **短信可靠性**：改进 PDU 解码、长短信合并、短信转发、短服务号码发送和发送记录。
- **安全与隐私**：限制 LuCI 后端参数，避免执行页面传入的任意命令；不向前端回显 PushPlus Token；测试数据使用合成内容。
- **移动端体验**：优化状态页和短信页的响应式布局，保证手机屏幕上的可读性和可操作性。
- **设备兼容性**：继续维护 RM520N、MT5700、NU313 等模组识别与差异化配置。
- **可维护性**：逐步替换难以调试的闭源辅助流程，补充测试、版本记录和独立仓库构建说明。

## 主要功能

- 蜂窝模组状态、信号和小区信息
- 分时网络制式、频段、频点和小区锁定
- SIM 卡槽、短信开关及模组参数管理
- 短信收件箱、PDU 解码、长短信合并
- 短信发送、内存发送记录及 PushPlus 转发
- 64 位累计流量、实时上传/下载速率和流量校准
- AT 命令调试与汇总日志
- 配置备份、恢复及自动任务
- MT5700 WebUI 集成

## 兼容性

- OpenWrt / ImmortalWrt LuCI
- 架构：`aarch64_cortex-a53`
- 主要设备：AK68、鲲鹏 C8-650/660/668 及相关定制设备
- 已涉及模组：RM520N、MT5700、NU313

不同硬件的 GPIO、LED、网口、模组固件和 AT 指令行为可能不同。移植到其他设备前，请先检查 `root/etc/init.d/` 和 `root/usr/share/modem-AK68/` 中的硬件相关脚本。

## 源代码与预编译组件

仓库保留了原始发行包中的部分 aarch64 预编译程序和动态库，例如 `atsd_tools`、`atsd_tools_cli.real` 和 `libbdinfo-AK68.so`。目前仓库内没有这些组件的完整可重建源代码，因此：

- 本仓库可以在目标 OpenWrt/ImmortalWrt 源码树中重新打包；
- 不能仅依靠本仓库重新编译上述预编译二进制组件；
- 对这些组件的进一步替换和可审计化，也是后续维护方向之一。

## 编译

将本仓库克隆到 OpenWrt/ImmortalWrt 源码树的 `package/` 目录：

```sh
git clone https://github.com/1220951120/luci-app-ModemATSD.git package/luci-app-ModemATSD
make menuconfig
make package/luci-app-ModemATSD/compile V=s
```

菜单中选择 LuCI 对应的 `luci-app-ModemATSD` 包。依赖包括：

- `luci-compat`
- `python3`
- `python3-requests`
- `bash`
- `curl`
- `flock`

## 安装

```sh
opkg install luci-app-ModemATSD_*.ipk
rm -f /tmp/luci-indexcache /tmp/luci-modulecache
```

> [!WARNING]
> 安装或升级包可能执行服务启动流程并重新检测、初始化模组。远程操作前请确认能够接受蜂窝网络短暂中断，并建议先备份当前配置。

## 测试

短信 PDU 与分时锁网调度测试可在源码树中运行：

```sh
python3 -m unittest discover -s tests -v
```

测试样本均为合成数据，不应提交真实手机号、短信内容、PushPlus Token 或设备备份文件。

## 近期改进

### 2.9-r23

- 将高级设置中原有的固定网络制式、频段和小区锁定全部合并到分时锁网规则，未命中规则时统一恢复自动网络。
- 分时规则拆分“网络制式”和“锁网类型”，新增允许/禁止小区重选与切换；支持用 `00:00` 到 `00:00` 表示全天锁网。
- 短信转发新增持久化 PDU 指纹基线，包升级或服务重启后不会把模组中已有短信当作新短信批量转发；状态文件只保存 SHA-256 指纹，不保存号码和正文。
- 本版本不兼容 r22 的 7 字段分时规则，升级后需要重新创建规则；旧的固定锁网 UCI 项也不再读取。

### 2.9-r22

- 将单一的“定时恢复自动网络”扩展为最多 8 条分时锁网规则，支持 LTE/5G-SA 锁频段以及锁 EARFCN/PCI。
- 未命中规则的时间自动解锁并恢复自动网络；相邻时间段在边界直接切换，不会中间触发自动网络。
- 新增紧凑的规则表格和折叠编辑区，支持跨午夜、重叠校验和移动端布局。
- 旧的 `auto_schedule_*` 配置不再读取且不自动迁移，升级后需要在高级设置中重新启用并添加分时锁网规则。

### 2.9-r21

- 修复打开“模组状态”时先显示上一次缓存数据的问题，状态接口现在会等待本轮采集完成后再返回。
- 移除模组状态页中与数据采集时间重复的“页面更新时间”栏。

### 2.9-r20

- 手机端收件箱改为卡片布局，短信正文独占整行。

### 2.9-r19

- 修复 `10010`、`10086` 等国内短服务号码被错误编码为国际号码的问题。
- 将模组返回 `OK` 的提示改为“已提交给模组”，避免误解为对端已经收到。

### 2.9-r18

- 将收件箱、发送短信、发送记录和 PushPlus 短信转发合并到“设备短信”页面。
- 发送短信改用独立 POST 接口，不再触发 CBI 的“保存设置”提示。
- 修复删除按钮换行，发送记录限制为内存中的最近 100 条。
- 转发状态接口不再返回完整 Token。

### 2.9-r17

- 使用 Python PDU 解码器解析 UCS2、GSM 7-bit 和短信时间。
- 自动合并长短信分段，清理异常的分段说明和发件人内容。
- 修复短信转发偶发空内容、空发件人和异常时间。
- 短信删除改为受限 POST 操作。
